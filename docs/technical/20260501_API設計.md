# API設計

作成者：泉知成  
作成日：2026年5月1日  
更新日：2026年5月8日

## 目的

Kuni-Musubi の MVP で必要な API を定義する。

MVP では、ログイン、記事保存、端末間同期、個人別閲覧履歴管理は実装しない。API は、記事表示、政党表示、カテゴリ表示、匿名集計イベント送信を中心に設計する。

## 基本方針

- 認証必須 API は作らない。
- ユーザー ID を前提にしない。
- localStorage の設定は、必要な条件だけリクエストに含める。
- 政治思想と閲覧履歴を個人単位で結合しない。
- 推薦 API は MVP では作らない。
- ホーム画面の記事表示は、`GET /articles` の絞り込みと並び替えで対応する。

## エンドポイント一覧

```txt
GET  /articles
GET  /articles/{article_id}
GET  /parties
GET  /parties/{party_id}
GET  /policy-categories
POST /analytics/onboarding
POST /analytics/article-event
```

## GET /articles

ホーム画面の記事カード一覧を返す。

### Query Parameters

```txt
party_id
category_ids
sort
limit
cursor
```

`sort` の候補:

```txt
latest
important
```

MVP では高度な推薦は行わず、以下の単純な表示制御に使う。

- 選択した政党の記事を先頭に表示する。
- 関心カテゴリで絞り込む。
- 政党タブで政党別の記事を表示する。
- 新着順または重要順で並べる。

```json
{
  "items": [
    {
```

## GET /parties/{party_id}

政党詳細画面で使う情報を返す。

### Response

```json
{
  "id": "party_001",
  "name": "自由民主党",
  "short_name": "自民",
  "leader_name": "string",
  "founded_year": 1955,
  "house_of_representatives_seats": 0,
  "house_of_councillors_seats": 0,
  "total_seats": 0,
  "color_hex": "#e60012",
  "ideology_summary": "string",
  "manifesto_summary": "string",
  "manifesto_promises": ["string"],
  "main_policy_categories": ["税金", "国防", "経済"],
  "official_url": "https://example.com",
  "latest_articles": [
    {
      "id": "article_001",
      "display_title": "string",
      "card_summary": "string",
      "published_at": "2026-05-01"
    }
  ]
}
```

`latest_articles` は最大 3 件までとする。政党詳細画面の UX が煩雑になる場合は、表示しない選択も可能とする。

## GET /policy-categories

オンボーディング、設定画面、記事タグ表示で使うカテゴリ一覧を返す。

### Response

```json
    {
      "id": "cat_001",
      "name": "税金",
      "slug": "tax",
      "display_order": 1
    }
  ]
}
```

## POST /analytics/onboarding

初回オンボーディング完了時に、匿名集計イベントを送信する。

### Request

```json
{
  "age_group": "20s",
  "selected_party_id": "party_001",
  "selected_party_status": "selected",
  "interest_category_ids": ["cat_001", "cat_002"]
}
```

`selected_party_status` の候補:

```txt
selected
none
unknown
skipped
```

個人を識別する ID は送信しない。

## POST /analytics/article-event

記事表示・クリック・詳細閲覧などの匿名イベントを送信する。

### Request

```json
{
  "article_id": "article_001",
  "event_type": "helpful_click",
  "surface": "article_detail"
}
```

`event_type` の候補:

```txt
impression
card_click
detail_view
source_click
helpful_click
unhelpful_click
```

`surface` の候補:

```txt
home
party_tab
party_detail
article_detail
```

このイベントは、支持政党や政治思想と個人単位で結合しない。

`helpful_click` は、記事詳細画面の「参考になった」ボタンが押されたことを表す。

`unhelpful_click` は、記事詳細画面の「そう思わなかった」ボタンが押されたことを表す。

これは公開いいねではなく、匿名・非公開のプロダクト改善用データとして扱う。記事カードや記事詳細画面に反応数は表示しない。

記事詳細画面の共有ボタン（X、LINE、リンクコピー）はクライアント側で共有 URL を生成する。共有操作を保存する API は MVP では作らない。
