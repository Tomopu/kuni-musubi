# LLM出力スキーマ設計

作成者：泉知成  
作成日：2026年5月1日  
更新日：2026年5月8日

## 目的

Kuni-Musubi の記事カード・記事詳細画面で使う表示内容を、LLM がどのような JSON で出力するかを定義する。

LLM はユーザーのリクエストごとに実行しない。バッチ処理で事前に実行し、結果を DB に保存する。

## 基本方針

- 出力は JSON とする。
- 表示タイトルはポジティブだが、誇張しすぎない。
- カード要約は 100 文字程度にする。
- 「何が Good News なのか？」として表示できるよう、前向きな進展を明確にする。
- 「我々の生活への影響」を生活者視点で書く。
- 「残る課題」を必ず出す。
- 世論・与野党からの評価は、攻撃的な言葉を避け、建設的な論点として整理する。
- カテゴリは既定のカテゴリ候補から選ぶ。
- サムネイル画像の生成は LLM に任せない。

## 出力スキーマ

```json
{
  "display_title": "string",
  "card_summary": "string",
  "related_parties": [
    {
      "party_name": "string",
      "party_short_name": "string",
      "relation_type": "primary"
    }
  ],
  "categories": ["string"],
  "positive_point": "string",
  "life_impact": "string",
  "remaining_issues": ["string"],
  "public_reactions_summary": {
    "government_or_ruling_party_view": "string",
    "opposition_view": "string",
    "public_opinion_or_expert_view": "string"
  },
  "source_summary": {
    "primary_source_url": "string",
    "source_type": "party_official",
    "source_name": "string",
    "published_at": "string"
  },
  "quality_flags": {
    "is_positive_news": true,
    "is_solution_oriented": true,
    "needs_human_review": false,
    "risk_notes": []
  }
}
```

## フィールド定義

### display_title

記事カードと記事詳細画面で使う表示タイトル。

方針:

- ポジティブで目に止まるタイトルにする。
- 政策や取り組みの前進を表現する。
- 事実以上に称賛しない。
- 煽り、断定しすぎ、人格評価を避ける。

### card_summary

記事カードで表示する 100 文字程度の要約。

方針:

- 専門用語を避ける。
- 1文から2文に収める。
- 詳しい課題や評価は記事詳細に回す。

### related_parties

記事に関連する政党。

`relation_type` の候補:

```txt
primary
mentioned
opposition_view
```

### categories

政策カテゴリ。

候補例:

- 税金
- 国防
- 金融政策
- 労働
- 社会保障
- 物価高
- 子育て
- 教育
- 移民
- 外交

記事カードでは最大 2 個まで表示する。3 個以上ある場合は「他2」のように UI 側で表示する。

### positive_point

「何が Good News なのか？」に表示する本文。

方針:

- 具体的な進展を説明する。
- 政党発表をそのまま称賛しない。
- 社会課題に対して何が前進したのかを書く。

### life_impact

「我々の生活への影響」に表示する本文。

方針:

- 生活者視点で書く。
- 税金、医療、雇用、物価、教育、安全保障などとの関係を説明する。
- 不確実な内容は断定しない。

### remaining_issues

「残る課題」に表示する配列。

観点:

- 財源
- 実現可能性
- 制度設計
- 副作用
- 対象から漏れる人
- 将来世代への影響

### public_reactions_summary

世論・与野党からの評価を整理する。

方針:

- 攻撃的な批判をそのまま載せない。
- 建設的な論点として言い換える。
- 情報が不足している場合は空文字または `情報不足` とする。

### quality_flags

人間による確認が必要かどうかを判定するための補助情報。

`needs_human_review` を true にする例:

- 出典が不明確
- ポジティブ判定が難しい
- 政党広報色が強すぎる
- 残る課題が抽出できない
- センシティブな外交・安全保障・差別関連トピック

## 禁止・注意表現

避ける表現:

- 断定的な人格評価
- 「完全解決」「歴史的快挙」などの過度な誇張
- 対立政党を貶める表現
- 政府や政党の無批判な礼賛
- 出典にない因果関係の追加

## 今後検討すること

- タイトル生成の具体的なプロンプト
- カテゴリ分類の候補リスト
- 人間レビューが必要な記事の基準
- 出力 JSON のバリデーション方法
- LLM 生成結果を管理画面で修正する運用

## dry-run での扱い

`batch.main --dry-run` では、LLM が生成した JSON をバリデーションするが DB には保存しない。保存されるのは通常実行時のみで、`articles`, `article_display_contents`, `article_sources`, `article_parties`, `article_categories` などに分解して登録する。
