## 概要

本ドキュメントは、バッチ処理に対して新しい政党サイト・RSS フィードを追加する手順、および Gemini API 向けプロンプトを調整する手順をまとめる。

操作方法・セーフティーガードについては [20260506_バッチ処理運用ガイド.md](20260506_バッチ処理運用ガイド.md) を参照すること。



## 政党・RSS フィードの追加手順

RSS フィードの設定は `batch/config/parties.yml` で一元管理している。コードを変更せずに追加できる。

### 手順

#### 1. フィード URL を事前確認する

追加前に以下を必ず確認する。

- `robots.txt` の確認: `https://[対象ドメイン]/robots.txt` を開き、`Kuni-Musubi-Bot` または `*` に対するアクセス制限がないことを確認する
- RSS フィードの存在確認: ブラウザで RSS フィード URL にアクセスして、`<rss>` または `<feed>` タグが返ることを確認する

#### 2. `batch/config/parties.yml` に追記する

```yaml
feeds:
  # --- 既存エントリは変更しない ---
  - url: "https://www.jimin.jp/news/rss/index.xml"
    source_name: "自由民主党"
    source_type: "party_official"
    max_items: 10

  # --- 新しい政党を末尾に追加する ---
  - url: "https://example-party.jp/news/rss.xml"  # 対象の RSS フィード URL
    source_name: "○○党"                             # DB に保存される政党名（Party.name と一致させる）
    source_type: "party_official"                   # 公式サイトなら party_official
    max_items: 10                                   # 1 回の取得上限（10 以下を推奨）
```

**フィールドの説明:**

| フィールド | 説明 | 例 |
|---|---|---|
| `url` | RSS/Atom フィードの URL | `https://xxx.jp/rss.xml` |
| `source_name` | 表示・DB 保存に使う政党名 | `日本維新の会` |
| `source_type` | 出典種別。公式なら `party_official` | `party_official` |
| `max_items` | 1 回の取得で処理する上限件数（10 以下推奨） | `10` |

**`source_name` の注意:** `steps/save.py` の `_resolve_party_id()` は `Party.name`（完全一致）で政党 ID を解決する。DB に登録されている名称と一致しない場合は保存時にスキップされるため、DB の `parties` テーブルに事前にレコードを追加しておくこと。

#### 3. `--fetch-only` で動作確認する

```bash
PYTHONPATH=..:../backend uv run python -m batch.main --fetch-only
```

新しいフィードから記事が取得できていることを確認する。

```
[fetch-only]  21. [○○党] タイトルが表示される
             URL: https://example-party.jp/news/12345.html
             公開日: 2026-05-06T10:00:00+09:00
             本文長: 512 文字
```

- 本文長が `0 文字` の場合: RSS がサマリのみでフル本文を提供していない（LLM 処理時はタイトルのみで処理される）
- URL が空欄になる場合: RSS の `<link>` タグ形式が非標準である可能性がある。`batch/scrapers/rss.py` の `fetch_rss()` でのパース処理を確認する

#### 4. `--dry-run` で LLM 出力を確認する

```bash
PYTHONPATH=..:../backend GEMINI_API_KEY=xxx uv run python -m batch.main --dry-run
```

新しい政党の記事が正しく LLM で処理され、`display_title` が適切に生成されることを確認する。



## Gemini API プロンプトの調整手順

プロンプトは以下の 2 ファイルで管理されており、Python コードを変更せずに調整できる。

| ファイル | 役割 |
|---|---|
| `batch/llm/prompts/system_prompt.txt` | LLM へのシステム指示（役割・禁止事項・出力方針） |
| `batch/llm/prompts/user_prompt.txt` | 記事ごとのユーザープロンプトテンプレート |

### システムプロンプト（`system_prompt.txt`）の調整

記事整形の方針を変更したいときに編集する。

**典型的な調整例:**

- カテゴリを追加したい: `## カテゴリ候補` のリストに追記する
- 禁止表現を追加したい: `## 禁止表現` のリストに追記する
- `needs_human_review` の判定基準を変更したい: 該当箇所の文章を書き換える

```
# 変更前
- **quality_flags.needs_human_review**: 出典不明・...の場合は true にする。

# 変更後（例: 福祉・医療トピックも対象に追加）
- **quality_flags.needs_human_review**: 出典不明・...・福祉・医療の場合は true にする。
```

### ユーザープロンプトテンプレート（`user_prompt.txt`）の調整

記事ごとに展開されるプロンプトの構造や、出力 JSON スキーマの説明を変更したいときに編集する。

**重要: プレースホルダの形式**

テンプレート内で以下の 2 種類の波括弧を区別すること。

| 記法 | 意味 | 例 |
|---|---|---|
| `{variable}` | Python の `.format()` で展開されるプレースホルダ | `{source_url}`, `{article_text}` |
| `{{literal}}` | JSON スキーマ内で波括弧をリテラルとして表示するためのエスケープ | `{{"display_title": "..."}}` |

JSON スキーマ部分を変更するときは、波括弧が `{{` `}}` でエスケープされていることを確認する。

```
# 正しい記述（JSON スキーマ内の波括弧はエスケープ）
{{
  "display_title": "string (300文字以内)",
  ...
}}

# 間違い（エスケープ不足で KeyError が発生する）
{
  "display_title": "string (300文字以内)",
  ...
}
```

### 調整後の検証

プロンプト変更後は必ず `--dry-run` で LLM 出力を確認する。

```bash
PYTHONPATH=..:../backend GEMINI_API_KEY=xxx uv run python -m batch.main --dry-run
```

出力の `display_title` や `card_summary` が意図どおりの内容になっているか確認する。



## RSS フィードを持たないサイトへの対応（将来の拡張）

現在の `fetch_rss()` は RSS 2.0 および Atom フィードのみ対応している。RSS フィードを公開していない政党サイトに対応する場合は、`batch/scrapers/` に新しいスクレイパーを追加し、`FetchResult` を返すインターフェイスに統一することで既存パイプラインに組み込める。

その際もセーフティーガード（`_get_with_retry` または `domain_rate_limiter`）を必ず使用すること。
