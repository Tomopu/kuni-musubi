## 結論

本プロダクトでは、メディアの批判的バイアスを排除しつつ、社会の多角的な視点を「建設的な論点」として取り込むため、以下の**ハイブリッド型のデータ収集・解析パイプライン**を採用する。

1.  **公式一次情報の取得（メイン）**: 各政党公式サイトを1日2回定期スクレイピング。ポジティブな実績を抽出の核とする。
2.  **多角的な視点の付与（サブ）**: 一般ニュースAPIから主要メディア（保守・中立・リベラル）の報道を収集。
3.  **LLMによる「感情の無毒化（サニタイズ）」**: 一般メディアの批判的・攻撃的なトーンを、LLMを用いて「今後の改善課題・議論のポイント」へと変換。ポジティブな空気感を維持しつつ、客観性を担保する。
4.  **計算コストの最適化**: 二次情報には「スニペット（概要）」のみを利用し、軽量LLMを活用することでトークン消費を抑制する。

## 2. メディア・バイアスと一次情報の重要性

一般メディアの報道には、以下の課題が存在するため、そのままの形で本プロダクトのメインソースにすることは不適当と判断した。

* **ネガティブ・バイアス**: ニュースの性質上、不祥事、対立、批判が強調されやすく、国民の「希望」を削ぐ要因となっている。
* **フィルターバブルの助長**: 各メディアの政治的立ち位置（社説等）により、特定の意見に偏った情報提供が行われやすい。

これを解決するため、**「政党が自ら発表した一次情報」**をベースとし、メディア情報はそれを補完する「客観的視点」として再定義する。

## 3. 一次情報取得の現状と課題

* **RSSフィードの欠如**: 自由民主党など一部を除き、ほとんどの政党がRSS配信を行っていない。
* **対策**: Pythonを用いた個別アダプター（スクレイパー）を実装。1日2回の低頻度バッチ処理により、相手サーバーへの負荷を抑えつつ確実に本文を取得する。

## 4. メディア論調の活用

メディアごとの政治的立ち位置を「対立」ではなく「多角的な視点」として活用する。

### メディア特性の分類

* **保守寄り**: 産経新聞（フジ）、読売新聞（日テレ）
* **中立**: 日本経済新聞（テレ東）
* **リベラル寄り**: 毎日新聞（TBS）、朝日新聞（テレ朝）

### 解析ロジック：批判の論点化

同じ題材に対し、右・左・中立の各スニペットをLLMに同時入力し、以下の処理を行う。
1.  **感情の濾過**: 「バラマキ」「無責任」といった攻撃的ワードを排除。
2.  **建設的抽出**: 「財源の確保」「将来世代への影響」といった「検討すべき論点」へと変換。
3.  **UI表示**: 1次情報の直下に「多角的な視点(世論・与野党からの評価)・今後の課題」として格納。

## 5. 実装効率とコスト最適化（トークン管理）

LLMのAPIコスト爆発を防ぐため、以下のエンジニアリング的工夫を施す。

* **スニペットの活用**: 二次情報（一般ニュース）はAPIから返却される「概要（スニペット）」のみを解析対象とし、本文取得を省略。

* **モデルの適材適所**:
    * フィルタリング/要約: GPT-4o-mini、Claude 3 Haiku等の軽量・低コストモデル。
    * 最終的な論点合成: 高性能モデル（GPT-4o等）を限定的に使用。

* **完全キャッシュ化**: バッチ処理で生成した要約済みデータをDBに保存。ユーザーのリクエストごとにLLMを走らせない「Zero-Runtime-LLM」構成。
