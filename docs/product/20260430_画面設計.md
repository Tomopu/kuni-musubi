# 画面設計

作成者：泉知成  
作成日：2026年4月30日  
更新日：2026年5月9日

## 目的

Kuni-Musubi の MVP で作成する画面と、各画面に表示する情報を定義する。

現時点では、高度な推薦機能よりも、政治に苦手意識のあるユーザーが「どの画面で、どの情報を、どの順番で見るのか」を明確にすることを優先する。

MVP では、ユーザーアカウント、ログイン、記事保存、端末間同期は実装しない。初回オンボーディングで選択された内容は localStorage に保存し、表示制御に利用する。オンボーディングと設定画面の回答はサーバーには送信しない。

## MVP で作成する画面

MVP で作成する画面は以下の 7 つとする。

1. 初回オンボーディング画面
2. ホーム画面
3. 設定画面
4. 記事詳細画面
5. 政党一覧画面
6. 政党詳細画面
7. このアプリについて画面

## 画面別設計

## 1. 初回オンボーディング画面

### 目的

- ユーザーの初期表示設定を作る。
- 支持政党、年代、関心テーマを localStorage に保存する。
- 政治的嗜好の入力に対する心理的抵抗を下げる。

### 質問項目

#### 年代

- 10代
- 20代
- 30代
- 40代
- 50代
- 60代以上
- 回答しない

年代は、まずは 10代・20代を見分けられる粒度を確保する。MVP では厳密な年齢は取得しない。

#### 支持政党

最初に「特になし」を置く。

選択肢の例:

- 特になし
- わからない
- 自由民主党
- 立憲民主党
- 日本維新の会
- 国民民主党
- 公明党
- 日本共産党
- れいわ新選組
- 参政党
- 社会民主党
- 日本保守党
- チームみらい
- その他

政党名は略称ではなく正式名称を基本とする。ユーザーが政治に詳しくない前提のため、「特になし」「わからない」を自然に選べる位置に置く。

#### 関心テーマ

複数選択可能にする。

選択肢の例:

- 税金
- 社会保障
- 賃上げ・労働
- 物価高
- 子育て
- 教育
- 国防
- 外交
- 移民・外国人政策
- エネルギー
- 地方創生
- 憲法
- 金融政策
- 環境
- 防災

### スキップ

すべての質問はスキップ可能とする。

スキップ導線は、画面下部などの目立ちにくい位置ではなく、ユーザーがすぐ認識できる位置に配置する。

例:

```txt
今はスキップする
```

### 注意書き

オンボーディング画面には、回答の扱いを明記する。

例:

```txt
回答は端末内に保存され、表示内容の調整に使われます。
サーバーには送信されません。
```

### 保存先

localStorage:

- 年代
- 支持政党
- 関心テーマ
- オンボーディング完了状態

## 2. ホーム画面

### 目的

- ユーザーが最初に政治ニュースへ触れる入口にする。
- 選択した政党や関心テーマに沿って、読みやすい「良いニュース」を表示する。
- 政党別に記事を切り替えられるようにする。
- 政党理解への導線を用意する。

### 表示構成

ユーザーが初回オンボーディングで支持政党を選択している場合、ホーム画面の最初にその政党名を表示する。

例:

```txt
自由民主党の良いニュース
```

その下に、該当政党にタグ付けされた記事カードを表示する。

「特になし」「わからない」「スキップ」の場合は、特定政党を先頭に固定せず、全政党横断の新着・重要ニュースを表示する。

例:

```txt
各政党の良いニュース
```

### 記事カード

記事カードの詳細は [20260430_記事カード表示設計.md](20260430_記事カード表示設計.md) で定義する。

記事カードには、最低限以下を表示する。

- サムネイル
- 記事タイトル
- 政党タグ
- カテゴリタグ
- 短い要約
- 公開日

サムネイルは MVP では必須にしない。表示する場合は、人間が設定したテキストベースサムネイルまたはカテゴリ別デフォルトを基本とする。

記事カードをタップすると、記事詳細画面に遷移する。

### 政党タブ

先頭セクションの下に、各政党をタブとして表示する。

タブを切り替えることで、政党ごとの記事一覧を確認できるようにする。

例:

- すべて
- 自由民主党
- 立憲民主党
- 日本維新の会
- 国民民主党
- 公明党
- 日本共産党
- れいわ新選組
- 参政党

### 政党一覧への導線

ホーム画面には、政党一覧画面への導線を配置する。

例:

```txt
各政党についての説明 >
```

### MVP での推薦方針

MVP では、高度な推薦アルゴリズムは重要視しない。

まずは以下の単純な表示制御にとどめる。

- 選択した政党の記事を先頭に表示する。
- 関心テーマに一致する記事を優先する。
- 各政党タブで政党別の記事を表示する。
- 必要に応じて、新着順・重要順で並べる。

「なぜこの記事が表示されたのか」の説明は、記事詳細画面またはカード内の補足として段階的に検討する。

## 3. 設定画面

### 目的

- 初回オンボーディングで選択した内容を編集できるようにする。
- localStorage の設定を書き換える。
- ユーザーがいつでも支持政党・関心テーマを変更できる状態にする。

### 編集項目

- 年代
- 支持政党
- 関心テーマ
- オンボーディングをやり直す
- 設定をリセットする

### データの扱い

設定変更は localStorage のみ更新する。オンボーディングと設定画面の回答はサーバーには送信しない。

## 4. 記事詳細画面

### 目的

- ニュースの詳細を確認できるようにする。
- 記事のポジティブな進展だけでなく、出典や残る課題も確認できるようにする。
- 政党広報やプロパガンダに見えないよう、検証可能性を担保する。

### タグ

記事には以下のタグを付与する。

- 政党名
  - 表示上は略称も利用可能
  - 例: 自民、立憲、維新、国民
- カテゴリ
  - 税金
  - 国防
  - 金融政策
  - 労働
  - 社会保障
  - 移民・外国人政策
  - など

### 表示項目

記事詳細画面の詳細は [20260430_記事詳細画面表示設計.md](20260430_記事詳細画面表示設計.md) で定義する。

表示項目:

- タイトル
- 関連政党
- カテゴリ
- 何が良いニュースなのか
- 我々の生活への影響
- 残る課題
- 世論・与野党からの評価
- 出典・一次情報リンク
- 参考になったフィードバック

「参考になった」は公開いいねではなく、匿名・非公開のプロダクト改善用フィードバックとして扱う。反応数は画面に表示しない。

「ふりがな」「説明をもっと詳しく」のような理解支援機能は、MVP では必須にしないが、政治に不慣れなユーザーの理解を助ける機能として今後検討する。

## 5. 政党一覧画面

### 目的

- ユーザーが各政党の全体像を把握できるようにする。
- 政党の勢力図や政策対立を視覚的に理解できるようにする。
- 政党詳細画面への入口にする。

### 表示項目

#### 議席数の可視化

- 衆議院の議席数割合
- 参議院の議席数割合
- 各政党の議席数

グラフは扇形グラフ、またはドーナツチャートを検討する。
各政党と議席数・テーマカラーは以下の通りである。()は衆議院と参議委員の合計議席数。

- 自民 (417) : #E60012 (ロゴ文字の鮮やかな赤)
- 維新 (55) : #98C93C (シンボルマークの黄緑)
- 国民 (50) : #F3C600 (右側背景の黄色)
- 中道 (49) : #0064A8 (背景のスタンダードな青)
- 立憲 (38) : #1A3382 (ロゴ文字の濃紺)
- 参政 (30) : #EB6100 (背景のオレンジ)
- 公明 (21) : #0D3370 (背景の深いネイビー)
- チみ (13) : #84D6CD (背景のミントグリーン)
- 共産 (11) : #DF1B22 (背景の赤 ※自民よりわずかに深みのある赤に調整)
- れい (7) : #E4007F (旗の鮮やかなマゼンタピンク)
- 保守 (2) : #0073CE (背景の明るい青)
- 社民 (2) : #00A1E9 (背景の水色)
- 減税 (1) : #3A4075 (背景の紫がかったインディゴ)
- 無所属 (13) : #999999 (中立的なグレー)

#### 政策対立の可視化

将来的には、各政党の政策に対する立場や対立関係を有向グラフで表示する。

ただし、政策対立の有向グラフは情報設計とデータ整備の難易度が高いため、MVP では必須にしない。

MVP では以下のような簡易表示から始める。

- 政策テーマごとの政党スタンス表
- 主要テーマごとの賛成・反対・慎重の整理
- 政党ごとの主要政策タグ

#### 政党詳細へのリンク

画面下部に各政党の一覧を表示し、タップすると政党詳細画面に遷移する。

表示項目:

- 政党名
- 略称
- 議席数
- 主要政策タグ
- 詳細を見る

## 6. 政党詳細画面

### 目的

- 1つの政党について、基本情報・理念・公約・関連ニュースを確認できるようにする。
- 政治に詳しくないユーザーでも、その政党が何を重視しているのか理解できるようにする。

### 表示項目

- 政党名
- 略称
- 代表者氏名
- 党成立年
- 所属議員数
- 衆議院議席数
- 参議院議席数
- 政治理念
- 主要公約
- 主要政策タグ
- 公式サイトリンク

### ニュースサムネイル

画面下部に、その政党に関連するニュースサムネイルを表示する案がある。

ただし、政党詳細画面から記事詳細画面へ遷移し、さらに戻る動線が増えるため、UX が煩雑になる可能性がある。

MVP では以下のどちらかを検討する。

#### 方針 A: ニュースサムネイルを表示しない

- 政党詳細画面は政党理解に集中させる。
- 関連ニュースはホーム画面の政党タブで確認してもらう。
- 画面遷移がシンプルになる。

#### 方針 B: 最新ニュースを最大3件だけ表示する

- 政党理解から具体的なニュースへ自然につなげられる。
- 表示件数を絞ることで画面の複雑化を抑える。
- 「もっと見る」はホーム画面の該当政党タブへ遷移させる。

MVP では、方針 B を採用する場合でも最大3件に制限する。

## 7. このアプリについて画面

### 目的

- Kuni-Musubi の意図と想いを伝える。
- 特定政党の宣伝や、政府・政党の無批判な肯定ではないことを明示する。
- ポジティブな政治ニュースを扱う理由を説明する。
- 出典、残る課題、複数視点を大切にする方針を伝える。
- ユーザーが安心して支持政党・関心テーマを設定できるようにする。

Kuni-Musubi は政治的嗜好や支持政党に触れるプロダクトであるため、「何を意図していて、何を意図していないのか」を明示する画面が必要である。

### 画面名

画面名は以下を第一候補とする。

```txt
このアプリについて
```

「想い」や「理念」だけでは主観的・重厚に見えやすいため、ユーザーが自然に開ける名称にする。

### 導線

主導線としては、フッターから遷移できるようにする。

ホーム画面でニュースを見る前に強く読ませると UX が重くなるため、ホーム上部に大きく出すのではなく、必要な人が確認できる位置に置く。

導線の候補:

- フッター
- 設定画面内
- 初回オンボーディングの注意書き付近

初回オンボーディングでは、以下のような補助リンクを置く。

```txt
このアプリの考え方を見る
```

### 表示項目

- Kuni-Musubi を作った背景
- 政治ニュースをポジティブに届ける理由
- 特定政党を応援するアプリではないこと
- 政府や政党を無批判に肯定するアプリではないこと
- 良いニュースだけでなく、残る課題や出典も表示する方針
- 一次情報や検証可能性を重視する方針
- 支持政党や関心テーマは端末内に保存されること
- 回答内容はサービス改善のため匿名集計される場合があること
- ユーザーの政治的嗜好をアカウントに紐づけないこと
- 目指すユーザー体験

### 表現方針

「愛国心を高める」と直接的に訴求しすぎると、ユーザーによっては警戒感を持つ可能性がある。

この画面では、以下のような表現を基本とする。

```txt
日本をより良くできると思えるきっかけを作る
```

また、政治を「怒りや対立だけの世界」ではなく、「生活と未来を良くするための手段」として捉え直せることを目指す。

### 注意点

- プロダクトの想いを語りすぎて、ニュース閲覧の邪魔にならないようにする。
- 特定政党・政府への誘導に見えないようにする。
- ポジティブニュースの意図と、プロパガンダではないことを明確に分けて説明する。
- データの扱いについて、初回オンボーディングと同じ方針で説明する。

## MVP で優先しないこと

- 高度な推薦アルゴリズム
- ログイン
- 記事保存
- 端末間同期
- 個人別の閲覧履歴管理
- 政党詳細画面での大量の記事一覧
- 政策対立の高度な有向グラフ
- 長文の思想表明を最初に読ませる導線

## 次に決めること

- 記事詳細画面に表示する項目
- 記事カードの具体的な UI
- 関心テーマの初期選択肢
- 政党一覧画面のグラフ表現
- 政党詳細画面のニュース導線を表示するかどうか
- このアプリについて画面の本文
