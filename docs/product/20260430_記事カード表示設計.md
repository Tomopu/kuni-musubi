# 記事カード表示設計

作成者：泉知成  
作成日：2026年4月30日  
更新日：2026年4月30日

## 目的

ホーム画面に表示する記事カードの情報設計を定義する。

Kuni-Musubi の MVP では、高度な推薦機能よりも、政治に詳しくないユーザーが「読みたい」と思える記事カードを作ることを優先する。

記事カードは、政治ニュースの重さや暗さを下げ、各政党・行政の建設的な取り組みを短時間で理解できる入口にする。

## 表示項目

記事カードに表示する情報は以下とする。

- サムネイル
- タイトル
- 要約
- タグ
- 日付

## サムネイル

サムネイルは、あるに越したことはないが、MVP では必須にしない。

理由:

- 既存ニュース画像や報道写真は著作権確認のコストが高い。
- 生成 AI で画像を作ると、生成コストが高くなる。
- 生成 AI 画像は AI っぽさが出やすく、ユーザー離れの原因になる可能性がある。
- 政治ニュースに対して画像の印象が強すぎると、内容よりも雰囲気で受け取られるリスクがある。

MVP でサムネイルを載せる場合は、AI に任せず、人間が設定したオリジナルのテキストベースサムネイルを利用する。

例:

- 政党カラー + 政策カテゴリ名
- 「税金」「国防」「労働」などのカテゴリ文字を中心にしたサムネイル
- 政党名とカテゴリを組み合わせたシンプルなサムネイル

画像がない記事は、カテゴリごとのデフォルトサムネイルを表示する。

## タイトル

タイトルは、できる限りポジティブかつユーザーの目に止まる表現にする。

ただし、誇張や煽りにならないようにする。特定政党の宣伝やプロパガンダに見えないよう、事実に基づいた範囲で「良いニュース感」を出す。

### 方針

- 政策・取り組みの成果や前進を伝える。
- 批判や対立ではなく、何が改善されたのかを中心に書く。
- ユーザーが「少し読んでみよう」と思える言葉にする。
- 功績を称えるようなトーンは使ってよいが、事実以上に持ち上げない。
- 記事詳細画面では、残る課題や出典も確認できるようにする。

### 例

```txt
高市内閣が石油不足による医療逼迫へ先手対策
```

```txt
チームみらいが議会の見える化アプリを開発
```

```txt
国民民主党が手取り改善に向けた新提案を発表
```

## 要約

要約は 100 文字程度を目安にする。

目的は、記事詳細を開かなくても「何が起きたのか」「なぜ良いニュースなのか」が大まかにわかることである。

### 方針

- 100 文字程度
- 専門用語を避ける
- 1文から2文に収める
- 政策の背景よりも、ユーザーが理解しやすい成果・変化を優先する
- 詳しい論点や残る課題は記事詳細画面に回す

## タグ

記事カードでは、タグを多く出しすぎない。

### 表示するタグ

- 政党名
- カテゴリ

政党名は必須とする。

カテゴリは最大 2 個まで表示する。3 個以上ある場合は、以下のように続きがあることを示す。

```txt
他2
```

### 政党名タグ

カード上では、略称を使ってよい。

例:

- 自民
- 立憲
- 維新
- 国民
- 公明
- 共産
- れいわ
- 参政
- 保守
- チみ

ただし、初回オンボーディングや政党詳細画面では正式名称を基本とする。

### カテゴリタグ

例:

- 税金
- 国防
- 金融政策
- 労働
- 社会保障
- 物価高
- 子育て
- 教育
- 移民
- 外交

## 日付

記事カードには日付を表示する。

表示形式:

```txt
2026年5月1日
```

日付は、記事の公開日または一次情報の発表日を使う。

## タップ時の動作

記事カードをタップすると、記事詳細画面に遷移する。

記事カード内に複数の細かいリンクを置かず、カード全体をタップしやすくする。

## いいね・参考になった

MVP では、記事カード上に「いいね」ボタンや「いいね数」は表示しない。

理由:

- 政党ごとの人気投票に見える可能性がある。
- 反応数が多い記事ほど正しいように見える可能性がある。
- 支持者動員や工作っぽさへの疑念が生まれやすい。
- ポジティブニュースの体験が、反応数競争に寄ってしまう。

一方で、どのようなニュース記事がユーザーにとって「参考になった」と感じられるのかは、プロダクト改善に重要である。

そのため、公開いいねではなく、記事詳細画面に **匿名・非公開の「参考になった」フィードバック** を置く方針とする。

記事カードでは、参考になった数やランキングは表示しない。

## MVP で優先しないこと

- AI 生成画像によるサムネイル
- 外部ニュース画像の自動取得
- サムネイルの完全自動生成
- カード内での長文説明
- カード内での残る課題の詳細表示
- 高度な推薦理由の表示
- 公開いいね数の表示
- 政党別・記事別の人気ランキング表示
